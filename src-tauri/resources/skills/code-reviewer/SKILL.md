# 代码审查工具

## 工作流程

1. 用户提供代码文件或目录
2. 执行审查：
   - 代码风格和规范
   - 潜在 bug 和安全漏洞
   - 性能问题
   - 最佳实践建议
3. 生成审查报告

## 输出格式

按严重程度分类：
- **Critical**: 必须修复的安全或逻辑问题
- **Warning**: 建议修复的潜在问题
- **Info**: 代码风格和最佳实践建议

每个问题包含：
- 文件路径
- 行号
- 问题描述
- 修复建议

最终给出整体评分和总结。
